Master Git & GitHub

The complete guide to version control and collaboration for DevOps engineers

What are Git and GitHub?

Git is a distributed version control system that allows developers to track changes in their code, collaborate with others, and manage project versions efficiently.

GitHub is a web-based platform that provides hosting for software development version control using Git. It offers all of Git's distributed version control and source code management functionality, along with its own features.

Version Control

Track and manage changes to your code over time

Collaboration

Work together with other developers seamlessly

Backup & Recovery

Never lose your work and easily revert changes

History of Git

2005

Creation of Git

Git was created by Linus Torvalds, the creator of Linux, for development of the Linux kernel. It was designed to replace the proprietary version control system called BitKeeper.

2008

GitHub Founded

GitHub was founded by Tom Preston-Werner, Chris Wanstrath, and PJ Hyett. It provided a web-based Git repository hosting service with a graphical interface.

2018

Microsoft Acquisition

Microsoft acquired GitHub for $7.5 billion, but committed to keeping it as an independent platform.

Today

Industry Standard

Git has become the de facto standard for version control, with GitHub hosting over 100 million repositories.

Essential Git Commands

git init

Initializes a new Git repository in the current directory.

$ git init

git clone

Creates a copy of an existing repository in a new directory.

$ git clone https://github.com/user/repo.git

git add

Adds file changes to the staging area.

$ git add filename      # Add specific file
$ git add .            # Add all changes

git commit

Records changes to the repository with a descriptive message.

$ git commit -m "Commit message"

git status

Shows the working tree status and staged changes.

$ git status

git branch

Lists, creates, or deletes branches.

$ git branch                 # List branches
$ git branch new-feature    # Create new branch
$ git branch -d branch-name # Delete branch

git checkout

Switches branches or restores working tree files.

$ git checkout branch-name    # Switch to branch
$ git checkout -b new-branch # Create and switch to new branch

git merge

Joins two or more development histories together.

$ git merge branch-name

git remote

Manages set of tracked repositories.

$ git remote -v                         # List remotes
$ git remote add origin https://github.com/user/repo.git

git push

Updates remote refs along with associated objects.

$ git push origin main

git pull

Fetches from and integrates with another repository.

$ git pull origin main

git stash

Temporarily shelves changes so you can work on something else.

$ git stash          # Stash changes
$ git stash pop      # Reapply stashed changes

git rebase

Reapplies commits on top of another base tip.

$ git rebase main

git cherry-pick

Applies the changes introduced by some existing commits.

$ git cherry-pick commit-hash

git log

Shows commit logs with various options for filtering and formatting.

$ git log
$ git log --oneline --graph  # Compact view with branch visualization

Real-World Scenario: Location Tracking Application

Let's explore how Git and GitHub would be used in developing a location tracking application that finds where users are using their devices.

1

Project Initialization

Initialize a Git repository for the location tracking application:

$ mkdir location-tracker
$ cd location-tracker
$ git init
$ git add .
$ git commit -m "Initial commit: Location tracker project structure"
2

Create GitHub Repository

Create a remote repository on GitHub and connect it to your local repo:

$ git remote add origin https://github.com/yourusername/location-tracker.git
$ git branch -M main
$ git push -u origin main
3

Feature Branch Workflow

Create separate branches for different features to enable parallel development:

$ git checkout -b feature/gps-integration
# Develop GPS integration functionality
$ git add .
$ git commit -m "Implement GPS location tracking"
$ git push origin feature/gps-integration
4

Code Review and Merge

Create a Pull Request on GitHub for code review before merging to main branch:

$ git checkout main
$ git pull origin main
$ git merge feature/gps-integration
$ git push origin main
5

Handling Bugs

When users report location accuracy issues, create a hotfix branch:

$ git checkout -b hotfix/location-accuracy
# Fix the accuracy algorithm
$ git add .
$ git commit -m "Improve location accuracy algorithm"
$ git push origin hotfix/location-accuracy

How Git/GitHub Helps This Project:

  • Version History: Track every change to the location algorithms
  • Collaboration: Multiple developers can work on different features simultaneously
  • Code Reviews: Ensure quality through pull requests before merging
  • Issue Tracking: Use GitHub Issues to track bugs and feature requests from users
  • CI/CD Integration: Automate testing and deployment when changes are pushed
  • Rollback Capability: Easily revert if a location tracking update causes issues
